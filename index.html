Open Soil Spectral Library

A short hands-on blog for working with the Open Soil Spectral Library (OSSL) and other spectral data.

Python package

If you want to process spectral data and try to model soil properties, I have developed a soil spectral processing package for the Open Soil Spectral Library (OSSL). The posts in this "Python package" section outline how to 1) clone or download the OSSL python package, 2) setup a virtual python environment in Anaconda, 3) install and setup the IDE Eclipse for running the python package and 4) run the python scripts; I then added a fifth post on how to add the regressor Cubist to the OSSL python package. If you are familiar with Python and its packages you only need to clone or download the python package and the example framework (1) and then edit the paths in the example framework to fit your local setup (4). If you want to add the Cubist regressor you also need to have a look at the last post (5) in this series.

All the following posts build a continuing series on how to process the OSSL data using the OSSL python package, ending up in Machine Learning (ML) models using spectral data as covariates for prediciting different soil properties.
